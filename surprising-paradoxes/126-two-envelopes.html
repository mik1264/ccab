The Flawed Reasoning

You pick Envelope A (call its amount X). You reason:

  • There's a 50% chance B has 2X (B is the bigger envelope)
  • There's a 50% chance B has X/2 (B is the smaller envelope)

❌ Wrong Calculation

E(B) = 0.5 × (2X) + 0.5 × (X/2)
E(B) = X + 0.25X = 1.25X

"B has 25% more! Always switch!"

✓ Correct Calculation

Let S = smaller amount
E(A) = 0.5 × S + 0.5 × 2S = 1.5S
E(B) = 0.5 × S + 0.5 × 2S = 1.5S

"Both envelopes have the SAME expected value!"

Where's the Flaw?

The error: X means different things in each branch!

When B is bigger, X represents the smaller amount. When B is smaller, X represents the larger amount. You can't use the same variable for both cases!

It's like saying: "I have a 50% chance of being taller than average, and 50% of being shorter. If taller, I'm above the average height. If shorter, I'm below the average height. Therefore my expected height is above average!" The logic is circular.

Historical Context

The Two Envelopes Problem has been a staple of probability theory discussions since at least the 1980s. It's related to the earlier "Exchange Paradox" and has generated hundreds of academic papers.

The paradox exposes subtle issues with:

  • Using the same variable for different conditional values
  • Improper prior probability distributions
  • The limits of expected value reasoning

Interestingly, there ARE some exotic probability distributions where you should always switch—but they require infinite money to set up, making them physically impossible!
