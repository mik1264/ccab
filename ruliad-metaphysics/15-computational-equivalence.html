Principle of Computational Equivalence

Three radically different formal systems — a cellular automaton, a tiny Turing machine, and a tag-system rewriter — all generate behavior of equivalent computational sophistication from absurdly simple rules. The ruliad is everywhere the same.

Cellular Automaton

Wolfram Rule 110 — universal
Step 0
Complexity (50‑step entropy) 0
8 neighborhood patterns → 01101110₂ = 110. Proven Turing‑complete (Cook, 2004).

Turing Machine

2‑state, 3‑symbol — universal class
Step 0
Complexity (50‑step entropy) 0
Two head states × three tape symbols. Rules: (A,0)→(1,R,B), (A,1)→(2,L,A), (A,2)→(1,L,A), (B,0)→(2,L,A), (B,1)→(2,R,B), (B,2)→(0,R,A).

Tag System

m = 2 string rewriter — universal class
Step 0
Complexity (50‑step entropy) 0
Each step: read first symbol, append its production, delete first m=2 symbols. Rules: 0 → 00, 1 → 1101. Seed: 10010.

Why this matters

The three substrates above have nothing in common structurally: one is a fixed 1D lattice updated in parallel, one is a single head walking a tape, one is a queue of symbols rewritten from the front. Their rules fit in a tweet. Yet each generates behavior of equivalent computational sophistication — none is reducible to a closed-form shortcut, and each is, in the appropriate sense, computation-universal.

"Almost all processes that are not obviously simple can be viewed as computations of equivalent sophistication." — Stephen Wolfram, A New Kind of Science

This is the Principle of Computational Equivalence, and it is the load-bearing claim under the ruliad: if every substrate that crosses a low threshold of richness ends up at the same computational ceiling, then the space of all possible computations — the ruliad — is in a deep sense everywhere the same. Physics, mathematics, and minds are different slices of one self-consistent object, viewed from different reference frames.
